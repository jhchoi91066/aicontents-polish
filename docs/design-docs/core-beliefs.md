# Core Beliefs

## 1. 규칙 기반 우선, AI 보조는 선택

정규식 기반 패턴 매칭이 핵심이며, AI(Gemini API) 기능은 optional peer dependency로 제공한다.
런타임에 외부 API 의존 없이 동작하는 것이 기본 원칙.

## 2. 한국어 AI 콘텐츠에 특화

한국어 LLM 생성 패턴(결론부터, 과장, 가짜 권위, 메타 내러티브)에 대한 가장 깊은 지식을 유지한다.
영어는 범용 지원이지만, 한국어는 전문가 수준의 감지를 목표로 한다.

## 3. 감지와 수정의 분리

`analyze()`는 읽기 전용 분석, `process()`는 수정을 포함.
사용자가 분석만 할지, 수정까지 할지 선택할 수 있어야 한다.

## 4. 확장 가능한 구성

`definePreset`과 `defineLocale`로 사용자 정의 확장을 지원한다.
`rules.disable`로 개별 규칙을 비활성화할 수 있다.

## 5. 성능은 사전 컴파일로

정규식은 모듈 로드 시 사전 컴파일하고, 런타임에서 재생성하지 않는다.
Cheerio DOM 파싱은 필요한 경우에만 수행한다.
