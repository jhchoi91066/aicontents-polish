# Planning Conventions

## 실행 계획 작성 규칙

1. 실행 계획은 `docs/exec-plans/active/`에 생성
2. 완료 시 `docs/exec-plans/completed/`로 이동
3. 각 계획에 포함할 내용:
   - 목표와 배경
   - 작업 분해 (체크리스트)
   - 의사결정 로그
   - 완료 기준

## 파일명 규칙

`{YYYY-MM-DD}-{짧은-설명}.md`

## 기술 부채

기술 부채는 `docs/exec-plans/tech-debt-tracker.md`에서 추적.
우선순위: High / Medium / Low
