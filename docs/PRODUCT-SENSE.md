# Product Sense

## 사용자 멘탈 모델

### 주요 사용자
- **블로그/콘텐츠 운영자**: LLM으로 생성한 글을 사람이 쓴 것처럼 다듬고 싶은 사용자
- **SEO 전문가**: AI 감지 도구를 우회하면서 품질을 유지하고 싶은 사용자
- **개발자**: 자체 CMS/파이프라인에 llm-polish를 통합하려는 사용자

### 핵심 가치

1. **신뢰**: 점수(0~100)가 콘텐츠 품질의 객관적 지표로 작동
2. **투명성**: 어떤 문제가 감지되고 어떻게 수정되었는지 report로 확인
3. **제어권**: 사용자가 규칙을 끄거나 커스텀 프리셋을 정의할 수 있음

## 우선순위 프레임워크

| 우선순위 | 기준 |
|:--|:--|
| P0 | 잘못된 수정 (원본 의미 훼손) → 즉시 수정 |
| P1 | 미감지 AI 패턴 추가 → 프리셋 패턴 확장 |
| P2 | 새 LLM/언어 지원 → 프리셋/로케일 추가 |
| P3 | 개발자 경험 개선 → API 편의성, 문서화 |

## 제품 결정 원칙

- **False positive보다 false negative가 낫다**: 잘못된 수정은 미감지보다 나쁘다
- **점진적 적용**: `analyze()`로 먼저 확인하고 `process()`로 수정
- **프리셋 독립성**: 각 LLM 프리셋은 독립적으로 동작해야 함
